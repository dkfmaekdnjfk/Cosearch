# 학습 세션 스킬

사용자가 모르는 개념을 **컨셉 문서**로 만들어 이해를 돕는다.
채팅으로 설명하지 않는다. 문서를 만든다.

---

## 절대 금지

- 채팅창에 개념 설명 ❌
- 뭘 모르는지 멋대로 판단 ❌
- 문서 없이 말로만 가르치기 ❌

---

## Step 1: 뭘 모르는지 확인

**반드시 먼저 물어본다.**

"어느 부분이 모르겠어?" 또는 "구체적으로 뭐가 안 와닿아?"
※ 사용자가 잘 설명하지 못할 경우 현재 맥락을 기반으로 제안해본다. 이해해야 하는 부분들을 쪼갠 후, 이미 이해된 부분을 확인받는 것도 방법이다. 다만 사용자가 부정하고 특수한 내용에 대한 설명을 요구할 경우 요구사항을 따른다.

체크리스트(`obsidian/00_Meta/Understanding_Checklist.md`)가 존재하면 참고해 ❓ 항목을 파악하되,
**사용자가 말한 것만 기준으로 삼는다.** 체크리스트로 추측하지 않는다.

---

## Step 2: 컨셉 문서 작성

저장 위치: `obsidian/20_Concepts/[주제].md`

### 컨셉 문서 구조

```markdown
---
title: "주제"
tags: [concept, 관련태그들]
date-created: YYYY-MM-DD
date-modified: YYYY-MM-DD
---

# 제목

관련 문서: [[연결 노트]]

---

## 1. [문제/배경] — 이게 왜 나왔는가

## 2. [핵심 내용] — 실제로 무슨 말인가

## 3. [의미/함의] — 이 연구에서 왜 중요한가

## 4. [한계 또는 열린 문제] — 아직 모르는 것

---

## 한 줄 요약
[딱 한 문장]
```

### 문서 작성 원칙

- **"새로 알게 된 것" 중심으로**: "이 개념은 X다"보다 "이것을 찾아서 Y를 알게 됐다"
- **구체적 예시 필수**: 추상적 설명만으로 끝내지 않는다
- **프로젝트 맥락 연결**: "우리 분석에서는..." / "리뷰어가 물으면..."
- **길이**: A4 1장 내외. 너무 길면 핵심이 묻힌다

---

## Step 3: 체크리스트 업데이트

파일: `obsidian/00_Meta/Understanding_Checklist.md`

- 파일이 존재하면:
  1. 해당 항목에 컨셉 문서 링크 추가: `[[컨셉 문서명]]`
  2. 상태는 그대로 ❓ 유지 — 사용자가 읽고 직접 ✅/⚠️로 바꾼다
- 파일이 없으면: 이 단계 건너뛴다

---

## Step 4: 보고

채팅 출력:
```
✅ 생성: 20_Concepts/[파일명].md
읽어봐. 이해되면 ✅로 바꾸면 돼.
```

다음 문서가 필요한지는 사용자가 읽은 뒤 판단하게 둔다. 미리 여러 개 만들지 않는다.

---

## 체크리스트 상태 기준

| 기호 | 의미 |
|------|------|
| ❓ | 모름 또는 미확인 |
| ⚠️ | 읽었지만 완전히 이해 못 함 |
| ✅ | 읽고 이해함 |

사용자가 ✅로 바꾸면 다음 개념으로 넘어간다.
