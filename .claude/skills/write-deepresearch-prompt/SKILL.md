# 딥리서치 프롬프트 작성 스킬

딥리서치 프롬프트 = **외부 AI 도구(Claude.ai deep research 등)에 줄 자기완결 문서**.
사용자가 이 파일을 직접 외부 도구에 붙여넣을 것이다.

---

## 절대 금지

- 서브에이전트로 조사 실행 ❌
- WebSearch / WebFetch / PubMed MCP로 직접 검색 ❌
- 채팅창에 프롬프트 전문 출력 ❌ (토큰 낭비)

---

## Step 1: 현재 프로젝트 상태 파악

아래 순서로 읽는다 (병렬 가능):

1. `obsidian/00_INDEX/PROJECT_STATUS.md` — 현재 논문 Claim, 병목, 실험 현황 파악
2. **배경 파일 확인**: `.claude/skills/write-deepresearch-prompt/references/` 폴더가 존재하는지 확인
   - 있으면 → 해당 파일을 읽고 Step 2에서 삽입
   - 없으면 → 사용자에게 묻는다:
     ```
     배경 섹션에 삽입할 연구 맥락 파일이 있나요?
     (없으면 PROJECT_STATUS.md 내용을 기반으로 작성합니다)
     ```

---

## Step 2: 프롬프트 구성

외부 AI는 우리 연구 맥락을 전혀 모른다. 반드시 포함할 것:

### 1. 배경 섹션
외부 AI가 올바른 논문을 찾으려면 연구 맥락을 충분히 알아야 한다.

- 배경 파일이 있으면 → 해당 파일 내용을 그대로 삽입 (줄이거나 요약하지 말 것)
- 배경 파일이 없으면 → PROJECT_STATUS.md의 논문 Claim + 현재 맥락을 기반으로 직접 작성:
  - 이 연구가 다루는 문제
  - 핵심 개념 및 용어 정의
  - 현재까지 확인된 것과 아직 미확인인 것

### 2. 탐색 대상 섹션

각 탐색 항목마다:
- 기존에 확인된 근거 (논문 제목·DOI 등 알고 있는 것)
- **정확히 무엇이 빠져 있는지** — 어떤 질문에 답하는 논문이 필요한가
- 유망한 검색 키워드

### 3. 보고 형식 섹션

외부 AI에게 아래 형식으로 보고하도록 요청한다:

```
각 논문에 대해:
- 제목, 저자, 연도, 저널, DOI/PMID
- 핵심 내용 (2~3줄)
- 탐색 항목과의 관련성
- 한계 또는 주의사항

확인되지 않은 사항은 명시적으로 "미확인"으로 표기할 것.
논문이 없으면 "해당 근거 없음"으로 보고할 것.
```
